description: Update README and ROADMAP before pushing
---
1. Analyze the current workspace, uncommitted changes, and recent unpushed commits to determine what new features or fixes have been implemented.
2. Update `ROADMAP.md` to mark newly completed tasks as done (change `- [ ]` to `- [x]`), or add any new planned features that came up during development.
3. Update `README.md` to reflect the latest state of the project. Ensure any new features, configuration options, or troubleshooting steps are properly documented.
4. Summarize the documentation changes and ask the user for approval.
5. If the user approves, stage and commit the changes to `ROADMAP.md` and `README.md`.
6. Offer to push the branch to the remote repository.
